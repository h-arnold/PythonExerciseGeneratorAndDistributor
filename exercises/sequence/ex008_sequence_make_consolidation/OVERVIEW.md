# Overview - Sequence Make Consolidation

## Prerequisites

- Understand that sequence code runs top to bottom.
- Know how to use `print()` for exact output.
- Be able to store values in variables and join strings together.
- Know that `input()` returns text.
- Be able to convert values with `int()`, `float()`, and `str()` when needed.

## Common misconceptions to address

- Forgetting that each graded cell runs on its own, so previous variables are not available.
- Printing the right words but with missing spaces, punctuation, or line breaks.
- Trying to do arithmetic with values from `input()` before converting them.
- Forgetting to convert a number back to text before joining it into a sentence.
- Mixing up when to use `int()` and when to use `float()`.

## Teaching approach and hints

- Start by modelling how to read the required output exactly before writing code.
- Encourage students to choose meaningful variable names instead of single letters.
- For the input exercises, ask students to say the type of each value aloud before calculating.
- If students get stuck, prompt them to solve the task one line at a time: prompt, input, calculation, final print.
- Keep hints small so this remains a genuine make activity rather than a disguised modify task.

## Suggested worked examples

- Build a welcome message from three word variables.
- Multiply a quantity by a price and print the result inside a sentence.
- Ask for one text input and print it inside a greeting.
- Convert a whole-number input with `int()` before multiplying.
- Convert a decimal input with `float()` before calculating a total.

## Expected progression

- Exercise 1: fixed output from variables only.
- Exercise 2: variables plus arithmetic and `str()`.
- Exercise 3: prompted text input and sentence output.
- Exercise 4: integer input, multiplication, and formatted final output.
- Exercise 5: float input, integer input, multiplication, and formatted final output.
